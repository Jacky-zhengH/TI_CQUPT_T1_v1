# Project Rules for Codex

## Project background

This is an STM32F407ZGT6 embedded project for the 2020 TI Cup amplifier nonlinear distortion research device.

The system uses:
- STM32F407ZGT6 as MCU
- AD7606 as external ADC
- HMI serial screen via USART1
- PC debug UART via USART3
- User code organized under User/app, User/bsp, User/alog

The competition target is to measure the output waveform uo of a transistor amplifier under five modes:
1. normal sine wave
2. top clipping distortion
3. bottom clipping distortion
4. both-side clipping distortion
5. crossover distortion

The software should acquire samples from AD7606, calculate Vpp and THD, and display the result on HMI.

## Editable areas

Codex may edit files under:

- User/app/
- User/bsp/
- User/alog/
- User/header.h only if necessary and after explaining why

## Do not modify

Do not modify CubeMX or HAL generated files unless I explicitly allow it.

Do not modify:
- Core/
- Drivers/
- Middlewares/
- startup files
- linker scripts
- .ioc file
- tim.c / gpio.c / usart.c / dma.c / adc.c generated by CubeMX

If peripheral configuration changes are needed, explain the required CubeMX settings instead of directly editing generated files.

## Layer rules

- bsp layer may call HAL functions and access hardware.
- app layer may call bsp and alog.
- alog layer must be pure algorithm code.
- alog must not call HAL functions.
- alog must not access GPIO, UART, HMI, DMA, or AD7606 directly.
- Interrupt callbacks should be short.
- Do not put printf or heavy calculation inside interrupt callbacks.
- THD calculation must run outside interrupt context.

## Current design assumptions

- AD7606 provides sampled output voltage data.
- HMI commands are sent through USART1.
- Debug messages are sent through USART3.
- The target waveform is around 1 kHz.
- THD is calculated using the 1st to 5th harmonic components.

## Coding style

- Keep existing Chinese comments when possible.
- Add clear Chinese comments for new code.
- Avoid large rewrites unless necessary.
- Preserve existing function names unless I ask to rename them.
- Prefer small functions and clear data structures.
- Before editing, summarize the files you plan to modify.
- After editing, summarize what changed and how to test it.
